# 08 模型增强数据与二次分析记录

## 1. 本轮目标

原始 UCI Online Retail II 数据能够支撑 RFM 和购物篮分析，但展示层的 Top 100 商品只有 8% 能直接命中全局关联规则，部分细分人群的推荐结果存在明显空白。本轮在不篡改原始事实数据的前提下，增加可追溯的模型预测场景数据，并建立“细分人群规则 → 全局规则 → 模型预测”的三级推荐路径。

## 2. 数据关系建模

生成器使用 `segmentAwareCooccurrenceV1` 模型，从真实订单中学习三类联合关系：

1. 商品关系：基于真实购物篮的共现次数、方向置信度和提升度选择锚点商品及搭配商品。
2. 客户关系：按最终 RFM 客群分层，保留真实客群规模、客户活跃频率及客户偏好商品的分布。
3. 金额关系：以客户或客群的真实购物篮金额中位数为预算中心，并结合真实商品单价、数量中位数生成正值交易金额。

生成过程固定随机种子 `20260721`，同一输入与配置可重复得到同一结果。邮费、折扣、坏账、人工调整等非商品编码不进入候选商品池。

## 4. 二次分析产出

增强输入共有 828,865 条交易明细，其中原始事实 768,865 条、模型预测 60,000 条。二次分析结果如下：

| 分析指标 | 结果 |
| --- | ---: |
| 客户数 | 6,348 |
| 订单数 | 50,275 |
| 商品数 | 4,631 |
| 销售额 | £22,479,921.96 |
| 平均订单金额 | £447.14 |
| RFM 快照 | 136,498 条 |
| 关联规则 | 9,353 条 |
| 规则漂移记录 | 9,472 条 |

Top 100 商品的全局正式规则覆盖率由增强前的 8% 提升到 92%。最终 API 在 `CHAMPIONS` 展示场景逐项验证 100 个商品：

| 推荐来源 | 商品数 | 说明 |
| --- | ---: | --- |
| `SEGMENT_RULE` | 78 | 直接命中当前 RFM 人群规则 |
| `GLOBAL_RULE_FALLBACK` | 15 | 人群规则缺失时使用全局真实+增强规则 |
| `MODEL_PREDICTION` | 7 | 正式规则缺失时使用高质量预测购物篮共现 |
| 空结果 | 0 | Top 100 实测非空率 100% |

模型预测回退设置三重质量门槛：置信度不低于 25%、提升度不低于接口请求值（默认 1.05）、共现购物篮不少于 `max(2, 预测订单数 × 0.2%)`。例如商品 `23166` 可预测搭配 `23298`，置信度 100%、提升度 30.60，覆盖 93 张预测购物篮，并在接口中明确返回 `sourceType=MODEL_PREDICTION`。

## 6. 复现命令

```powershell
# 生成模型增强数据
.\.venv\Scripts\intellibasket.exe augment-data `
  --input data\processed\hive\basket_items.tsv `
  --rfm-input outputs\analytics\rfmCustomers.csv `
  --output data\processed\hive\basket_items_augmented.tsv `
  --synthetic-output outputs\augmentation\syntheticTransactions.csv `
  --manifest-output outputs\augmentation\manifest.json `
  --target-rows 60000 `
  --generation-batch-id AUG-20260721-02

# 二次分析
.\.venv\Scripts\intellibasket.exe run-analytics `
  --input data\processed\hive\basket_items_augmented.tsv `
  --output outputs\analytics

# 回灌服务库
.\.venv\Scripts\intellibasket.exe load-serving-data `
  --input outputs\analytics `
  --augmentation-input outputs\augmentation
```

## 7. 展示口径

驾驶舱中的原始历史指标和模型预测必须分开展示。增强数据摘要卡明确显示批次、模型、预测行数、预测金额和置信度；推荐卡显示数据依据与来源类型。模型数据用于教学演示、情景推演和减少界面空白，不应被解释为真实发生的 UCI 客户交易。
