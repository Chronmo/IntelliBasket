# RFM与购物篮算法验证记录

> 里程碑：第二阶段 / 分析算法  
> 验证日期：2026-07-21  
> 正式输入：Hive `dws_basket_item` 导出数据

## 1. 实现内容

- 五分位动态RFM评分。
- 八类可运营客户分群。
- 月末RFM快照与客群迁移。
- 全部客户FP-Growth关联规则。
- 各RFM客群独立FP-Growth关联规则。
- 支持度、置信度、提升度、覆盖订单数与综合排序。
- 前后年度关联规则漂移分类。
- 经营总览、月度趋势与商品排行输出。

## 2. 正式输入

算法没有绕过数仓读取原始Excel，而是通过下列链路获得输入：

```text
Hive dws_basket_item
    ↓ INSERT OVERWRITE DIRECTORY
HDFS /data/intellibasket/exports/basket_items
    ↓ getmerge
data/processed/hive/basket_items.tsv
```

实际输入为768,865条“订单-商品”聚合明细，对应36,969张有效购物篮、5,878名客户和4,631种商品。

## 5. 分群创新

关联规则不只在全部订单上运行。最终规则范围为：

| 范围 | 规则数 |
| --- | ---: |
| 全部客户 | 100 |
| 重要价值客户 | 100 |
| 重要保持客户 | 100 |
| 重要挽留客户 | 100 |
| 新客户 | 100 |
| 潜力客户 | 100 |
| 重要发展客户 | 87 |
| 流失风险客户 | 42 |
| 一般客户 | 36 |

规则数量少于100不代表失败，而是该客群只有较少规则同时满足支持度、置信度、提升度和覆盖订单要求。

## 6. 规则漂移

使用相邻两个年度窗口比较全局规则，得到193条漂移记录：

| 漂移状态 | 数量 | 解释 |
| --- | ---: | --- |
| NEW | 93 | 当前年度新出现的强规则 |
| DROPPED | 93 | 上一年度存在、当前年度不再满足阈值 |
| GROWING | 5 | 提升度增长超过0.1 |
| DECLINING | 2 | 提升度下降超过0.1 |

漂移结果可用于区分稳定搭配和阶段性组合，防止把一次性热点长期用于营销。

## 7. 性能与质量

- 真实算法全流程耗时约54秒。
- 生成5,878名客户的最终RFM结果。
- 生成按月动态快照与迁移记录。
- 生成765条关联规则与193条漂移记录。
- 算法经营总览销售额为£17,743,429.18，与源数据、Hive DWS和ADS一致。
- 自动化测试覆盖评分、分群汇总、关联规则、漂移与数据接入。
