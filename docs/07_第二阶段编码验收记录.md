# 第二阶段编码验收记录

## 1. 阶段结论

第二阶段已完成从真实零售订单到可交互营销决策驾驶舱的全链路编码。项目不依赖伪造演示数据：HDFS、Hive、Python 分析、MySQL、FastAPI、Vue/ECharts 与 Docker 均已在本机环境实际运行并验收。

## 2. 完成范围

### 数据工程

- 将 UCI Online Retail II 两个工作表转换为标准 CSV，共 1,067,371 条明细；
- 上传 HDFS 并完成 Hive ODS、DWD、维度、DWS、ADS 六层 SQL；
- 通过有效客户、非取消单、正数量、正价格等规则得到 805,549 条有效分析明细；
- 以 `invoiceNo` 作为可靠购物篮边界，得到 36,969 张有效购物篮。

### 算法分析

- 生成 98,557 条多时间切片 RFM 客户快照；
- 最新快照把 5,878 位客户划分为 8 类客群；
- 使用 FP-Growth 生成全局及分客群规则 765 条；
- 创新增设规则漂移分析，识别 93 条新兴、93 条消失、5 条增强与 2 条减弱规则；
- 形成客群迁移、月度经营指标、高价值商品和业务概览等标准输出。

### 服务与展示

- 以事务方式幂等导入 MySQL 8.0；
- 提供 11 个 `/api/v1` 标准接口、OpenAPI 文档、请求 ID、统一响应与错误信封；
- Vue 端完成经营总览、动态 RFM、分群购物篮和营销策略中心四个页面；
- 默认策略场景可从真实订单中生成 6 个可解释商品组合；
- 提供 MySQL、FastAPI、Nginx 三服务 Compose 部署。

## 3. 代码规范落实

- Python、TypeScript、Vue 中的变量、函数、方法和 JSON 字段使用 lowerCamelCase；
- 类、Vue 组件和类型使用 PascalCase；
- 常量使用 UPPER_SNAKE_CASE；
- SQL 文件名与 Python 模块名遵循各自生态的 snake_case 约定；
- API 使用名词资源、HTTP 方法语义、`/api/v1` 版本和分页上限；
- Ruff、TypeScript strict、数据库约束和自动化测试共同保证质量。
